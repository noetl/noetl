# Python package retirement list for 5.0.0

NoETL 5.0.0 scopes the `noetl` PyPI package to the CLI only. The Python
server, worker, gateway-adjacent helpers, tool implementations, outbox,
projector, and database runtime modules were removed from the wheel. Those
behaviors now live in Rust components deployed through the NoETL ops
playbooks.

Removed package trees:

- `noetl.core`
- `noetl.database`
- `noetl.outbox`
- `noetl.projector`
- `noetl.server`
- `noetl.tools`
- `noetl.worker`

Removed transitional modules:

- `noetl.cli_wrapper` (subprocess wrapper around a binary)
- `noetl.install` (post-install binary copier)
- `noetl.main` (retired Python command entrypoint)
- `noetl.claim_policy` (legacy Python policy helper)

Kept CLI-relevant Python modules:

- `noetl.__init__`
- `noetl._cli`
- `noetl._native` (PyO3 extension generated by maturin)

Known limitation:

- The default wheel does not enable `noetl-tools/duckdb-integration`, so
  local playbooks using `kind: duckdb` return an explicit opt-in error.
