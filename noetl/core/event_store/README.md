# `noetl.core.event_store`

Backend-neutral **event store port** with a reference PostgreSQL adapter.
This package owns the durable append-only event log that the rest of the
distributed runtime (projector, outbox, replay API) reads from.

## Status

- **Port:** `EventStore` Protocol with `append` / `read` ([ports.py](ports.py))
- **Reference adapter:** `PostgresEventStore` against `noetl.event` ([postgres.py](postgres.py))
- **Phase:** delivers the runtime side of v2 distributed-runtime spec phase 2;
  port-and-adapter shape from spec phase 5. See
  [`repos/docs/docs/features/noetl_distributed_runtime_spec.md`](https://github.com/noetl/docs/blob/main/docs/features/noetl_distributed_runtime_spec.md)
  for the authoritative design.

## Interface contract

```python
class EventStore(Protocol):
    async def append(
        self,
        stream_id: str,
        events: list[EventRecord],
        *,
        expected_version: Optional[int] = None,
    ) -> int: ...

    async def read(
        self,
        stream_id: str,
        *,
        from_version: int = 1,
        limit: int = 1000,
    ) -> list[dict[str, Any]]: ...
```

- `append` is transactional and uses optimistic concurrency. Pass
  `expected_version` to enforce a CAS check against the current stream head.
  Raises `ExpectedVersionConflict(stream_id, expected_version, actual_version)`
  on mismatch. Pass `None` to accept any current version.
- `append` returns the last written `stream_version` (the new stream head).
- `read` returns events in `stream_version` ascending order.

## `EventRecord` envelope

The envelope is the wire format for every event. Field order is stable; the
envelope checksum is computed over a canonical JSON serialization (sorted
keys, ISO-8601 timestamps with `Z` suffix, UTC normalized) excluding
`event_id`.

| Field | Type | Notes |
|---|---|---|
| `event_id` | `int` | Snowflake ID generated by the adapter at append time. Not part of checksum. |
| `tenant_id` | `str` | Multi-tenant scope. Default `"default"`. |
| `organization_id` | `str` | Sub-tenant scope. Default `"default"`. |
| `execution_id` | `int?` | NoETL playbook execution this event belongs to. |
| `stream_id` | `str` | Aggregate stream identifier вҖ” ordering is strict within one stream. |
| `stream_version` | `int` | Assigned by adapter; monotonic per `stream_id`. |
| `aggregate_id` / `aggregate_type` | `str?` | Optional DDD-style aggregate identity. |
| `event_type` | `str` | Required. Drives projection routing and subject derivation. |
| `schema_name` / `schema_version` | `str / int` | Default `noetl.<event_type>` / `1`. |
| `event_time` / `ingest_time` | `datetime` | Producer time vs. server-assigned `now()`. |
| `producer` | `str?` | Free-form producer identity. |
| `causation_id` / `correlation_id` / `idempotency_key` | `str?` | Tracing + dedupe metadata. |
| `payload_ref` | `dict?` | JSONB reference to a payload in the (future) payload store. Off-event-stream payload >256KB lives here. |
| `result` / `meta` | `dict` | JSONB. Small inline payload + meta annotations. |
| `status` / `node_name` | `str?` | Lifecycle status + producing pod/node name. |
| `envelope_checksum` | `str` | SHA-256 hex of the canonicalized envelope (sans `event_id`). Computed by `EventRecord.envelope()`. |

See `canonical_event_checksum()` for the exact serialization rules.

## Postgres adapter behavior

[`PostgresEventStore`](postgres.py) writes to the `noetl.event` table. Per
append call:

1. Acquires `pg_advisory_xact_lock(hashtext(stream_id))` вҖ” per-stream
   serialization without locking the whole table.
2. Computes `current_version = COALESCE(max(stream_version), 0) WHERE stream_id = ?`.
3. If `expected_version` is set and differs from `current_version`, raises
   `ExpectedVersionConflict`.
4. For each event in the batch: increments version, generates a snowflake
   `event_id` via `noetl.snowflake_id()`, inserts the envelope, and (if the
   event mirror is enabled) enqueues a row into [`noetl.outbox`](../../outbox/README.md)
   for asynchronous fan-out to NATS вҖ” both writes commit atomically.
5. After commit, triggers a best-effort `publish_outbox_batch()` drain
   capped by `NOETL_EVENT_STORE_OUTBOX_DRAIN_LIMIT` (default 100). Failures
   here are logged and swallowed вҖ” the outbox worker is the authoritative
   drainer.

The `noetl.event` schema is managed by NoETL's main migrations, not by this
adapter.

## Configuration

| Env var | Default | Effect |
|---|---|---|
| `NOETL_EVENT_MIRROR_ENABLED` | `false` | Enable the transactional outbox enqueue on every append. Must be `1`/`true`/`yes`/`on`. |
| `NOETL_EVENT_STORE_OUTBOX_DRAIN_LIMIT` | `100` | Max outbox rows the post-commit best-effort drain attempts to publish per append call. |

When `NOETL_EVENT_MIRROR_ENABLED` is unset/false, appends never touch the
outbox вҖ” useful for unit tests and single-process dev loops.

## Usage

```python
from noetl.core.event_store import EventRecord, ExpectedVersionConflict, PostgresEventStore

store = PostgresEventStore()

next_version = await store.append(
    stream_id="execution/12345",
    events=[
        EventRecord(
            event_type="action.completed",
            stream_id="execution/12345",
            execution_id=12345,
            aggregate_id="action/abc",
            aggregate_type="action",
            result={"ok": True},
        )
    ],
    expected_version=4,  # CAS вҖ” must match current head, else raises
)

events = await store.read("execution/12345", from_version=1, limit=1000)
```
